# Generated API Tests

Auto-generated by [`openapi-test-gen`](https://github.com/vinipx/tooling-helpers).

## Setup

Install dependencies:

```bash
npm install
```

## Running Tests

```bash
npm test
```

## File Structure

| File | Description |
|------|-------------|
| `config.js` | Runtime configuration — `BASE_URL` (env-var overridable) |
| `endpoints.js` | Frozen object mapping semantic names → `{ method, path }` |
| `validators.js` | Ajv-compiled validators for 2xx response schemas |
| `*.test.js` | Vitest test stubs, one file per API tag |

## Before Running

1. To target a different environment, set the `BASE_URL` env variable — no regeneration needed:
   ```bash
   BASE_URL=https://staging.myapi.com npm test
   ```
2. Replace every `PLACEHOLDER_*` value in test URLs with real test data.
3. If an `--auth-header` was injected, update the `AUTH_HEADERS` constant in each test file.
4. For endpoints with a request body, add the body to the `fetch` call (see the TODO comment).
